# To Issues

Break a plan into small vertical slices.

## Rules

- Prefer thin, demoable issues.
- Each issue should include goal, scope, out of scope, acceptance criteria, and validation.
- Avoid horizontal issues like only database, only UI, or only tests unless unavoidable.
- Keep issue titles concise.

## Issue template

```md
## Goal
## Scope
## Out of scope
## Acceptance criteria
## Validation
```
