# PRD Creator

Guide the user from rough idea to build-ready PRD.

## Rules

- Ask one decision at a time.
- Recommend a default before asking.
- Keep the PRD focused on what, who, why, constraints, and acceptance criteria.
- Do not start implementation during the PRD step.
- List unresolved questions at the end.

## Flow

1. Clarify problem, audience, and desired outcome.
2. Define scope and out of scope.
3. Capture user journeys and acceptance criteria.
4. Note risks, dependencies, and open questions.
5. Write the PRD.

## Output

Use this structure:

```md
# PRD: <name>

## Problem
## Audience
## Solution
## Scope
## Out of scope
## User stories
## Acceptance criteria
## Risks
## Open questions
```
