# Grill Me With Docs

Challenge a plan before it becomes code.

## Rules

- Ask one question at a time.
- Give a recommended answer with each question.
- If the answer may exist in docs or code, inspect them first.
- Track decisions, risks, and unresolved questions.
- Stop when the plan is clear enough to execute.

## Good questions

- Which existing decision could this contradict?
- Which domain term is ambiguous?
- What must keep working?
- What is reversible?
- What validation proves this is correct?
